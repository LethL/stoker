# ADR 0003: Абстракция майнера за интерфейсом MinerDriver

**Дата:** 2026-06-18
**Статус:** Принято

## Контекст

Stoker должен снимать телеметрию майнера и (частично) им управлять. Майнеров
много (WildRig Multi, T-Rex, lolMiner, gminer, TeamRedMiner…), у каждого свой
API, формат JSON, порт, набор команд. Версии майнеров часто меняются (автор
сейчас использует wildrig-multi v0.48.4), а схемы их API не документированы и
могут дрейфовать между релизами.

Research 0002 зафиксировал: **№1 причина смерти open-source аналогов
(Minera, cryptoGlance, MiningMonitor) — жёсткая привязка к одному
майнеру/монете/эпохе.** Research 0003 подтвердил, что даже у одного майнера
(WildRig) API недокументирован и его форму надо снимать с живого бинаря.

## Решение

Ввести **границу `MinerDriver`** (Python `Protocol`) в `agent/src/stoker_agent/
miners/`. Драйвер инкапсулирует всё майнер-специфичное и наружу отдаёт
**нормализованную Pydantic-модель `MinerStats`**, а не сырой JSON майнера.

Минимальный контракт:

```python
class MinerDriver(Protocol):
    async def start(self, cfg: MinerConfig) -> None: ...   # spawn процесса
    async def stop(self) -> None: ...                      # kill процесса
    async def telemetry(self) -> MinerStats: ...           # нормализованная модель
    async def is_running(self) -> bool: ...
```

**Реализуем только `WildRigDriver`** (первый риг). Другие драйверы — аддитивно,
по мере реальной надобности, без изменений в сервере/UI/метриках.

## Обоснование

- `Protocol` = structural typing, аналог TS-интерфейса: подходит любой класс с
  нужными методами, наследование не требуется; mypy проверяет на CI.
- Нормализованный `MinerStats` изолирует дрейф API майнера в одном месте:
  переименовали поле в новой версии WildRig — правим один драйвер.
- Defensive parsing: поля, способные уехать между версиями, — `Optional` с
  дефолтами; неизвестные поля игнорируются (Pydantic по умолчанию). Новая
  версия майнера не роняет коллектор.
- Lifecycle (start/stop) живёт в драйвере, т.к. у остановленного майнера нет
  API — запуск возможен только через процесс/systemd (см. [[0005-miner-api-localhost-binding]]).

## Последствия

- Сервер, Prometheus-экспозиция и UI не знают марку майнера — оперируют только
  `MinerStats`. Добавление lolMiner/gminer = новый драйвер, ноль изменений выше.
- Конфиг майнера на риге — TOML; драйвер рендерит из него командную строку
  конкретного майнера (для WildRig — CLI-флаги, не JSON-конфиг).
- Перед релизом драйвера обязателен живой дамп API реального бинаря как
  pytest-fixture (для WildRig — `curl http://127.0.0.1:<port>/`).

## Альтернативы и почему отвергнуты

- **Хардкод под WildRig напрямую:** ровно та привязка, что убила аналоги;
  смена майнера = переписывание половины агента.
- **Полноценная плагинная система на N майнеров сразу:** over-engineering на
  этапе M0–M4; нарушает дисциплину scope. Граница — да, зоопарк реализаций
  заранее — нет.
