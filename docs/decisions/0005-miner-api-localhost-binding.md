# ADR 0005: API майнера биндится на localhost, контроль брокерится агентом

**Дата:** 2026-06-18
**Статус:** Принято

## Контекст

Майнеры выставляют локальный HTTP/сокет API для телеметрии и управления.
Research 0002/0003 выявили серьёзную проблему безопасности:

- **T-Rex** не разделяет read-only `/summary` и привилегированные
  `/control`/`/config` — открытый порт = возможность shutdown и чтения/подмены
  кошелька/пула.
- **WildRig Multi** — HTTP-JSON API (XMRig-style, у HiveOS порт 60050) **без
  аутентификации**, bind по умолчанию недокументирован, read-only режима нет.

Если такой API доступен по сети, любой в LAN/WAN может выключить майнер или
прочитать конфиг.

## Решение

1. **Агент запускает майнер с API, биндингованным только на `127.0.0.1`**
   (loopback). Порт майнера наружу не выставляется никогда.
2. **Единственный клиент API майнера — сам агент** на том же хосте.
3. **Любое управление снаружи** идёт через аутентифицированный WebSocket-канал
   `server↔agent`; агент транслирует команду сервера в локальный вызов API
   майнера или в управление процессом.
4. Считаем API майнера **zero-trust и без auth** независимо от конкретного
   майнера.

## Обоснование

- Майнерские API исторически без auth и без разделения прав — выставлять их
  нельзя.
- Агент уже работает на риге и владеет процессом майнера — естественный
  брокер и единственная точка доступа.
- WS agent→server (агент дозванивается наружу) не требует входящих дыр в
  файрволе ригов и даёт аутентифицированный канал для команд.

## Последствия

- В конфиге запуска майнера агент всегда форсит loopback-bind (для WildRig —
  явный `--api-port` на 127.0.0.1; проверить наличие bind-флага на реальном
  бинаре, иначе файрволить порт).
- Start/stop майнера — через процесс/systemd (см. [[0003-miner-driver-abstraction]]),
  не через API (у остановленного майнера API нет).
- Аутентификация/авторизация WS-канала agent↔server — обязательна (детали в
  отдельном ADR при проектировании протокола в M2d).
- Pause/resume через API WildRig — best-effort, точный endpoint проверяется на
  живом бинаре.

## Альтернативы и почему отвергнуты

- **Выставить API майнера в сеть и ходить с сервера напрямую:** прямой security
  risk (shutdown/кража конфига), исключено.
- **Web-SSH / удалённый шелл к ригу:** запрещено правилами проекта без
  отдельного ADR с серьёзным обоснованием; для управления майнером избыточно.
- **Полагаться на read-only режим API майнера:** у WildRig его нет, у T-Rex —
  тоже; нельзя строить безопасность на несуществующей фиче.
