# ADR 0007: UI на Vue 3.5 (разворот с React 19)

**Дата:** 2026-06-18
**Статус:** Принято (заменяет выбор React в исходном CLAUDE.md)

## Контекст

Исходно (CLAUDE.md, секция «Стек») фронтенд был зафиксирован на **React 19** с
обоснованием: у автора сильный React-бэкграунд, не размазывать когнитивную
нагрузку — новое в проекте — Python/DevOps/метрики, фронт держать в знакомом.

При ревизии перед стартом M1 автор пересмотрел приоритет. Вводные:

- Автор последние ~3 года работает только на React (преимущественно v18).
- React 19 в нашем сетапе (чистый **Vite CSR-SPA** + embedded Grafana, без
  Next.js/SSR) почти не раскрывается: его ключевые новинки (Server Components,
  Actions, `use()`) живут в SSR-фреймворках. В CSR React 19 ≈ React 18 → роста
  почти не даёт.
- Зимой автор прошёл курс по **Vue 3.5** (от работодателя). Знания из курса без
  боевого проекта быстро выветриваются.
- Проект — **самообразовательный**, дедлайнов нет; явная цель — развитие автора
  как специалиста.

Таким образом изменился **вес критериев**: не «доставить быстро + фокус на
backend», а «вырасти как специалист». При этом весе фронт-фреймворк — главный
рычаг роста, а не нейтральная деталь.

## Решение

UI пишем на **Vue 3.5** (Composition API, `<script setup lang="ts">`).

Сопутствующий стек (замена React-эквивалентов):

| Назначение | Было (React) | Стало (Vue) |
|---|---|---|
| Фреймворк | React 19 | **Vue 3.5** |
| Клиентский стейт | Zustand | **Pinia** |
| Роутинг | TanStack Router | **Vue Router** |
| Server-state/data fetching | TanStack Query | **TanStack Vue Query** (`@tanstack/vue-query`) — сохраняем |
| Сборка / язык / стили | Vite / TS / Tailwind | без изменений |

## Обоснование

- **Технического победителя нет:** для дашборда (список ригов, статусы, embedded
  Grafana, позже flight sheets) React и Vue эквивалентны. Решает цель обучения.
- **Второй фреймворк = реальная ширина:** отделяет универсальное от
  React-специфичного, повышает версатильность и найёмопригодность (Vue силён на
  многих рынках).
- **Закрепление начатого:** проект конвертирует курс Vue 3.5 в устойчивый навык,
  иначе он деградирует.
- **Низкий риск нагрузки:** кривая Vue мягкая, у автора уже есть экспозиция, а
  фронт стартует на M3 — после Python-тяжёлых M1/M2, поэтому нагрузка
  разносится по времени, а не складывается.
- **Vue Query сохраняет** ранее принятое решение по server-state (есть Vue-адаптер).

## Последствия

- Правки CLAUDE.md: стек, «Почему именно так», «Чего НЕ используем»
  (Redux→Pinia, React Router→Vue Router), структура `web/` (`router/`, Pinia
  stores, `main.ts`), упоминания React 19 в целях/диаграмме.
- M1-скелет web: `pnpm create vite web --template vue-ts`; Vue Router/Pinia/
  Vue Query подключаются на M3 (UI-веха), не в M1.
- Фронт пойдёт медленнее привычного React — принято осознанно (нет дедлайнов).
- Меньше прямого переиспользования паттернов Beszel (он на React) — минорно.
- Chart-кандидаты для inline-виджетов переосмыслены под Vue (uPlot/vue-chartjs/
  unovis) — решение на M3 (дополнение к Grafana, не замена).

## Альтернативы и почему отвергнуты

- **Остаться на React 19:** быстрее и привычнее, но в CSR-SPA не даёт нового
  навыка — противоречит пересмотренной цели (рост). Отвергнуто автором осознанно.
- **React 19 + изучать его глубоко (RSC/Actions):** эти фичи требуют SSR
  (Next.js), что вне scope (мы CSR + Grafana). Нерелевантно нашему сетапу.
- **Svelte/Solid:** ещё незнакомо и без начатого обучения — большая нагрузка без
  задела; Vue выигрывает за счёт уже пройденного курса.
